# AGENTS.md

このリポジトリ（`ninja-habits`）で作業するエージェント向けのメモです。
フロントエンド（React + TypeScript + Vite）と API を扱います。

## 文書の読み方

- 共通方針とコーディングルールはワークスペース側 `../AGENTS.md` と `../docs/`（`../docs/rule-index.md` の該当作業の行）から選ぶ
- このリポジトリ固有の設計は `docs/`（API 設計・DB 設計・セキュリティ・提案）に置く

## 目的

- `ninja-habits` の実装や設計を進める
- 提案、決定事項、次に試したい案を失わないように残す

## 作業ルール

- 新しい提案や改善案が出たら `docs/proposals.md` に追記する
- 実装前に前提や未確定事項があれば `docs/proposals.md` に残す
- 大きな方針変更をしたら、理由も短く記録する
- まだ決まっていないことは断定せず、候補として書く

## ドキュメント配置

- 提案メモ: `docs/proposals.md`
- API 設計: `docs/api-design.md` / DB 設計: `docs/db-design.md` / セキュリティ: `docs/security.md`
- 必要に応じて設計メモや要件メモを `docs/` 配下へ追加する

## 記録フォーマット

以下のような形で簡潔に残すこと。種別は `決定` / `未決` / `提案` を使い分け、確定済みと未確定案を混ぜない。

```md
## YYYY-MM-DD

- 提案:        # 種別に応じて 決定: / 未決: に置き換える
- 理由:
- 次のアクション:  # 未決・提案で残課題があるとき
```
